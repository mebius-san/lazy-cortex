# lazycortex-specs — changed since publish

**Status**: changed — local version `3.0.0` is ahead of the published `2.1.1` in `CHANGELOG.public.md`. Run `/pub.publish` to ship the pending changes.

## Highlights since 2.1.0

- **Breaking — product docs flattened to the product root.** The per-product `docs/` subfolder is gone: `design.md` and `tech.md` now live directly at the product root next to the product folder-note, and `docs/docs.md` is removed. Product slugs `design` and `tech` are now reserved to avoid colliding with those files. Existing vaults must move `<product>/docs/design.md` and `<product>/docs/tech.md` up to `<product>/` and delete the empty `docs/`. `spec.doctor` flags a stray `docs/` folder and a reserved slug.
- **Breaking — every folder-note now opens with an automatic `# Summary`.** The asset status note's body is three plugin-owned first-level sections — `# Summary` (a short LLM-written précis of what the feature/change/bug is about), `# Gates`, and `# History` — each tagged `#protected/spec/<region>` so other tools leave them alone; the old `# <slug> — status` title line is gone. Category, product-root, and the requests inbox notes carry a `# Summary` with a one-line précis plus a small live statistics line (asset counts by lifecycle position, or request counts by status) that the plugin keeps current automatically as assets change. Operator prose lives below the protected Summary. Existing vaults re-author note bodies into the new shape (no automatic conversion).
- **Breaking — spec content now lives under a configurable `spec.vault_root` (default `specs/`).** Subsystem folders and the shared `requests/` inbox sit under `<repo>/specs/` instead of the repo root, so spec content no longer clutters the repository top level; the `requests/` inbox is always created and committed so it is tracked even when empty. Set `spec.vault_root` in `lazy.settings.json` to override (use `"."` to keep content at the settings directory). Existing vaults move their subsystem folders and `requests/` under the chosen root.
