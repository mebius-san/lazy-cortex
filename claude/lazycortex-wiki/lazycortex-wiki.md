# lazycortex-wiki — pending changes

**Status**: changed — `plugin.json` version `1.3.0` is ahead of the last published release `1.2.0`; the change ships on the next `/pub.publish`. Re-run `/wiki.install` (or `/wiki.configure`) after updating to pick up the new scope filter default.

## Highlights since `1.2.0`

- **Per-scope review filter** — each `wiki.scopes.<id>` now takes an optional `filter` (a frontmatter predicate of the same shape as a routine's `filter` block); `/wiki.configure` seeds `review_active: {not_in: [true]}` by default, so a document opened for review drops out of the wiki entirely — not curated, not indexed, not linked — and rejoins automatically once review closes. The same predicate is seeded into the `wiki.scan` routine as an early cut, and is enforced on every path (`process-file`, `relink-all`, `/wiki.relink`, `doctor`).
- **See-also is a protected H1 section** — the curated See-also block is now `# See also` with an owner tag `#protected/wiki/see-also` on its first line (the HTML markers stay inside). The tag marks it a protected cross-plugin region, so `lazycortex-review` (and any other file-mutating tool) preserves it byte-for-byte during a review while the wiki keeps managing the links between the markers; link changes go through the `wiki_pinned_links` / `wiki_unrelated_links` pins, not the block body.
