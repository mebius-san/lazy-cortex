# lazycortex-wiki — changed since publish

**Status**: changed — version `1.6.0` is ahead of published `1.5.2`; the changes below await the next `/pub.publish`.

## Highlights since 1.5.2

- Deleting a wiki node no longer leaves dangling links behind: a new `wiki.scan-deletes` daemon routine watches for deleted files and runs the deterministic `prune-node` CLI, which drops every See-also line pointing at the removed node, rebuilds the scope's `topics.md`, and commits — no curator dispatch needed. `/wiki.relink` runs the same prune for nodes deleted since its anchor, so the cleanup also works without the daemon.

- Fixed a repair bug in `wiki.doctor --apply`: dropping a broken See-also line (or refreshing a stale gloss) stripped the `- ` list prefix from the surviving lines, corrupting the See-also section. Kept lines now stay valid markdown list items.
