# lazycortex-core — changed since publish

**Status**: changed — version `5.16.1` is ahead of published `5.16.0`; the changes below await the next `/pub.publish`.

## Highlights since 5.16.0

- Chained and flag-form commits are no longer missed by the commit gates: `git add … && git commit … && git push`, `cd x && git commit …`, and `git -C dir commit` / `git -c k=v commit` now all trigger the commit-recorder hook (feeding `.logs/commits.jsonl`) and the pre-publish secret scan — previously both fired only when the Bash command literally began with `git commit`, so most real commits slipped through silently.

- A commit that succeeded inside a chain whose later segment failed (e.g. a rejected `git push`) is no longer lost: on a non-zero exit code the recorder now checks whether `HEAD` is fresh and unrecorded before skipping, so only genuinely failed commits stay out of the feed.
