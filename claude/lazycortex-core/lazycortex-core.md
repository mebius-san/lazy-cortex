# lazycortex-core — changed

**Status**: changed — version `5.11.0` is ahead of published `5.10.0`; unpublished work is pending the next `/pub.publish` run.

## Highlights since 5.10.0

- Dead-job scan no longer misclassifies a finished job: when the pump process dies first, a grace window lets the in-flight response land before DEAD is considered, and jobs whose response is already on disk are finalized as done/failed instead of being marked dead.
- Every daemon-dispatched agent now resolves an explicit model: the built-in doctor is a regular config-resolved expert seeded by the installer (entry + sonnet tier), a dispatch that resolves no model is recorded loudly in the error ledger, the runtime preflight fails unpinned experts with a concrete pin-model fix, and finished doctor jobs no longer block the next doctor dispatch for days.
- Two non-interactive maintenance agents for cross-project rollout: `lazy-core.autosetup` re-runs the install chain for one repo (applying derivable steps, reporting question-gated ones), and `lazy-core.autocheckup` runs the checkup passes and applies only mechanically derivable fixes — both commit their changes in the target repo.
