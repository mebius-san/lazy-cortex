## Highlights since 5.8.2

- Background expert spawns now run under an idle-stdout watchdog: a frozen `claude -p` stream — no first token, or silence mid-response — is detected after `daemon.stream_idle_timeout_sec` (default 90 s) of no stdout, its process group is killed, and the spawn is re-tried from a fresh session up to `daemon.stream_max_retries` (default 3) times. A transient network/API freeze that used to burn the whole 30-minute routine timeout is now absorbed within seconds; each stall is recorded to `.logs/lazy-core/runtime/stream-stalls.jsonl`. Both settings are optional with built-in defaults, so existing installs need no changes.
