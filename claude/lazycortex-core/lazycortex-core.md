## Highlights since `5.4.0`

- **Login-equivalent daemon environment for headless hosts** — the runtime supervisor can now give the daemon the operator's login environment so `claude -p` authenticates and the `claude` binary resolves on hosts that launchd/systemd start without a login shell. Two opt-in `daemon.supervisor` options: `login_shell` (re-exec through a login shell for the full token + PATH) and `env_files` (source named env files for just the token). Both off by default — existing installs behave exactly as before.
