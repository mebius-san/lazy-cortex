# lazycortex-core — pending changes

**Status**: changed — `plugin.json` version `5.7.0` has not been published yet (last recorded release: `5.6.0`).

## Highlights since `5.6.0`

- **Expert spawns are hermetic by default, with an opt-in per-expert MCP allow-list** — the daemon now spawns every expert with `--strict-mcp-config`, so ambient operator MCP servers (which block on initialization in a headless, TTY-less context and let the job die on its timeout) are never inherited. An expert that genuinely needs specific servers declares `mcp_config` (a path or list of MCP-config files) under its `lazy.settings.json[experts]` entry; only those load. The new `/lazy-runtime.preflight` skill validates that every routine's expert is actually launchable — static config checks plus a trivial-prompt launch probe that catches hanging or auth-blocked MCP servers — and applies operator-confirmed fixes.
