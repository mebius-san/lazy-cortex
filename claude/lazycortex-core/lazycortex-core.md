# lazycortex-core — changed

**Status**: changed — version `5.12.0` is ahead of published `5.11.2`; unpublished work is pending the next `/pub.publish` run.

## Highlights since 5.11.2

- Several daemons can now share one machine without stepping on each other's metrics: the runtime learns to enumerate every installed daemon from its supervisor units, hands each new checkout the next free metrics port automatically, and publishes a Prometheus file_sd targets file covering all of them — so an external Prometheus is pointed at one file once and picks up every future daemon by itself. A taken port no longer crash-loops the daemon: it records a named incident (including who holds the port) and keeps dispatching without metrics. Dashboard labels default to the readable `local-<folder>` instead of a hash, and the install skill offers metrics enablement with the port stored per-machine.
- The agent-model defaults now carry a tier for the new tester expert from lazycortex-experts, so fresh installs seed it without a wizard round. The planner expert now defaults to Opus, entries for agents that no longer exist (the old review dispatcher / process-file pair) are gone, and the defaults list is kept alphabetically sorted.
