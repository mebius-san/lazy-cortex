# lazycortex-core — patch pending

**Status**: patch — version `5.11.2` is ahead of the published `5.11.1`; run `/pub.publish` to ship.

## Highlights since 5.11.1

- Runtime daemon now reconciles the `daemon_halted` metric with `state.json` on every iteration, so a halt gauge no longer stays pinned in Grafana after the halt clears (auto-recovered dirty tree or external `/lazy-runtime.recover`); the cumulative halt counter is unaffected.
- The non-interactive `lazy-core.autosetup` and `lazy-core.autocheckup` agents now decide which install chains apply to a target repo from that repo's own enabled plugins (its `settings.json` / `settings.local.json`), instead of the machine-wide plugin registry — so a repo enabling only a couple of plugins no longer receives the install chains of every plugin present anywhere on the host.
