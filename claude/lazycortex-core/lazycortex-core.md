# lazycortex-core — pending changes

**Status**: changed — `plugin.json` version `5.6.0` has not been published yet (last recorded release: `5.5.0`).

## Highlights since `5.5.0`

- **Expert-spawn sandbox isolated from the interactive session** — the daemon's `claude -p` expert spawns now load their sandbox from a daemon-owned `.runtime/sandbox.settings.json` passed via `--settings`, instead of `.claude/settings.local.json`. The interactive session in the same checkout is no longer sandboxed (so the operator's `git push` over SSH works again), while spawns stay confined; permission scope still lives in `settings.local.json` and reaches the spawn via the cwd merge. Re-run `/lazy-core.install` to relocate the block — the install step migrates any legacy `sandbox` key out of `settings.local.json` automatically.
- **Inbox routines dispatch the file path, not its inlined content** — an inbox file now stays in the inbox as the single source of truth (parity with the `git` / `md-scan` routines), and only its path is handed to the expert. Non-text inputs (PDFs, images, binaries) are no longer corrupted by UTF-8 decoding. Each tick reconciles finished work: a succeeded job drains its input, while a failed job leaves it parked as a dead-letter for the operator to triage. Experts may move or delete the input only as their last action on success (new clause in the expert-runtime contract).
