# lazycortex-core — changed since publish

**Status**: changed — version `5.15.0` is ahead of published `5.14.0`; unpublished commits under `claude/lazycortex-core/`.

## Highlights since 5.14.0

- Skills that need a live interactive session (loaded MCP tools) now declare `requires_live_session: true`; the headless autosetup agent skips them as `live-session-only` instead of failing every run — `lazy-guard.allow-mcp` carries the flag, interactive `/lazy-core.setup` still runs it normally.
- A dirty working tree is no longer invisible on dashboards: the daemon exports a `dirty_tree` gauge for the whole time it silently pauses routine dispatch over uncommitted changes, and token series now carry an `expert` label so spend is attributable to the actual consumer (`money.update-runner`, …) instead of lumping under the pump.
- Install-time file sync (rules, authoring templates, the runtime shim) is now triaged by a deterministic script instead of agent judgment: absent targets are copied, present ones byte-compared, and only genuinely diverged files reach the merge/conflict decision — an "already-current" verdict now requires the script's receipt, so it can no longer be declared by impression.
