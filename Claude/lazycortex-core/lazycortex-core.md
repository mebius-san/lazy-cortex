# lazycortex-core — changes since `2.0.1`

**Status**: changed — version `2.1.0` is ahead of the published `2.0.1` (top of `CHANGELOG.public.md`). Run `/pub.publish` when ready.

## Highlights since `2.0.1`

- The runtime daemon now accepts `--plugin-dir <path>` (repeatable) so a dev-vault operator can point it at live plugin sources instead of routing every routine through the user's `~/.claude/plugins/cache/`. The resolver matches each path's `.claude-plugin/plugin.json` `name` and falls back to the cache when no dev-mode match is found, so existing setups keep working untouched.
- Daemon iterations now SKIP entirely when the working tree is dirty at the top of an iteration, instead of running routines and halting on whatever they did. Routines like `lazy-review.tick` read source files and commit a new version — running them on a dirty tree would read the operator's WIP edits, generate output from that mid-edit state, and commit over the operator's work under a bot identity. Silent skip lets the daemon resume cleanly the next polling cycle once the operator commits or whatever held the tree dirty finishes. Post-routine dirt is now unambiguously a routine contract violation (tree was clean at start) and still halts the daemon for operator review.
- The agent / protocol reference resolver now reads dev-plugin paths from a `LAZYCORTEX_PLUGIN_DIRS` env var (exported automatically when the daemon starts with `--plugin-dir` flags). Subprocess routines like `lazy-review tick` and `lazycortex-core expert-pump-once` inherit the env, so spawned experts resolve `<plugin>:<name>` against the live source tree instead of failing with `plugin not in cache` mid-loop.
