## Highlights since 3.0.0

- **Expert aspects** — experts can declare `aspects[]` in `lazy.settings.json[experts]`; each aspect is a composable behavior layer the pump lists in the user-message prompt alongside protocols. See `references/lazy-core.expert-aspects-contract.md`.
- **Expert arguments** — experts can declare `arguments{}` (JSON-serializable values, keys `^[a-z][a-z0-9_]*$`) that flow through to every job's `config.json` and render as `- argument: <name> = <json-value>` lines in the prompt.
- **Memory subsystem** — `.memory/<expert>/` (tracked in git) gives persona-marked experts long-term memory. Four new skills: `/lazy-memory.write`, `/lazy-memory.index`, `/lazy-memory.reflect`, `/lazy-memory.mark-persona`. Plus the `memory-reflect-all` CLI verb for periodic batch consolidation.
- **Prompt-label simplification** — the legacy `- protocol contract:` label is now `- protocol:`, sitting in parallel with `- aspect:` and `- argument:`.
- **lazy-expert.dispatch-job fix** — the standalone dispatch skill now writes `config.json`, including aspects + arguments. Previously direct-dispatched jobs were silently skipped by the pump.
