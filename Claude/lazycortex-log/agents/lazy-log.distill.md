# Distill Commits → Changelog

Turn raw commit entries into human-readable functional prose in `./docs/changelog.md`.

## Input

None — the agent reads state from the filesystem.

## Process

### Step 1: Read state

- Read `./.logs/commits.jsonl` (raw commit entries from the `lazy-log.commit-recorder` hook). Each line is a JSON object: `{sha, date, author, branch, subject, body, files, insertions, deletions}`.
- Read `./docs/changelog.md`. Look for the marker `<!-- lazy-log: last-distilled-sha = <sha> -->` to find the last processed commit. If the marker is missing or says `none`, process from the beginning.
- If `commits.jsonl` doesn't exist or is empty, tell the user there's nothing to distill and exit.

### Step 2: Determine commits to process

Walk `commits.jsonl` in order. Skip everything up to and including the last-distilled-sha. Collect the rest as "pending commits".

If there are no pending commits, tell the user the changelog is up to date.

### Step 3: Read diffs

For each pending commit, run `git show --stat --format=""  <sha>` to see what files changed and their stats. Do NOT read full diffs unless a commit's subject is unclear — the commit message + file list is usually enough.

### Step 4: Write functional entries

For each pending commit, write a **1-3 sentence** description focused on what changed from the **user's perspective**:

Good:
- "Renamed cortex skills to the `lazy-core.*` / `lazy-guard.*` / `lazy-repo.*` namespace. Users now invoke `/lazy-core.audit` instead of `/cortex.audit`."
- "Added a post-commit hook that records every commit to `.logs/commits.jsonl` for later changelog generation."

Bad (too technical):
- "Renamed 5 files in `~/Dropbox/Dotfiles/claude/skills/cortex.*` to `Claude/lazycortex-core/skills/lazy-*.*`, updated 11 cross-references across SKILL.md files, fixed 2 log path references."

Bad (too vague):
- "Updated some skills."

Rules for writing:
- If the commit subject already reads as functional prose (e.g., "fix: use correct hook protocol for settings guardian deny"), use it nearly verbatim with minimal rephrasing.
- If the commit subject is `chore:`, `refactor:`, or similar internal work, write a 1-sentence note and flag it as internal: prefix with `(internal)`.
- If multiple commits form a coherent unit (same day, related subjects), you MAY group them under one bullet with a date header.
- Include the SHA in backticks at the end of each bullet: e.g., `(a2739ff)`.

### Step 5: Format and prepend to changelog

The changelog uses a loose date-grouped format:

```markdown
# Changelog

Functional descriptions of changes to this repo, generated by the `lazy-log.distill` agent from `.logs/commits.jsonl`.

The most recent changes appear at the top.

<!-- lazy-log: last-distilled-sha = <newest-sha-processed> -->

## 2026-04-16

- Added the `lazycortex-log` plugin with commit recorder, changelog distillation, and change-history recall (`a2739ff`, `78e1cca`).
- (internal) Fixed settings guardian hook to use the correct deny protocol (stdout + exit 0) (`f2283b0`).

## 2026-04-15

- Migrated cortex skills into the `lazycortex-core` plugin; renamed to `lazy-core.*` / `lazy-guard.*` / `lazy-repo.*` (`8ab5c73`).
- Added the `publish` skill that syncs this repo's `Claude/` dir to the public repo (`06dc71c`).

(older entries below, unchanged)
```

Insert new entries **below the marker and above the most recent existing date header** (reverse chronological — newest first).

Update the marker to the newest SHA just processed.

### Step 6: Report

Tell the user:
- How many commits were distilled
- Range of dates covered
- Last-distilled-sha moved from `<old>` to `<new>`
- Path to the updated file

## Guidelines

- **Don't re-write existing entries** — only append new ones. If the user wants to regenerate history, they should say so explicitly (and you'd start over from `none`).
- **Never auto-commit the changelog** — the user commits it themselves if/when they want.
- **Stay short** — this is a changelog, not a blog post. If a commit really needs more detail, put the SHA and let the reader `git show` it.

## Logging

Log to `./.logs/claude/lazy-log.distill/YYYY-MM-DD_HH-MM-SS.md` per the logging rule (include `git_sha` in frontmatter).
Use `Bash(mkdir -p ...)` then `Write` tool (never chain).
